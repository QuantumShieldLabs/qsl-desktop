Vault & Security

Unlock protection

The escalating unlock delay is always on: the first two wrong attempts are free, then a 5-second delay doubles per failure, capped at 5 minutes. It defends this app's unlock path on this device; an offline copy of the vault file is defended only by your passphrase strength.

Erase vault after failed attempts

Off by default. If armed, reaching the limit permanently erases the vault. A guest — or a child at the keyboard — can trigger this. Arm it only if you understand the consequence.

Idle autolock

On by default (15 minutes). Applies to the main window and settings; the setup wizard is exempt.

Danger zone

Destroying the vault requires your current passphrase and permanently erases it through the core's confirmed-destroy path.
